## Inclusion Criteria

The following criteria are applied:

1. `meta_year` must be non-missing and within 2014 to 2022,
2. `meta_system` must be non-missing and match esophagus, stomach, intestine, liver or pancreas,
3. `meta_ops` must be present,
4. `meta_age >= 18`,
5. Duplicate cases removed using `meta_case` (first kept after sorting by `meta_incision`).

## Preprocessing Summary

- **Categorical variables** → ordinal (numeric) encoding or one-hot-style string encoding, based on the presence / absence of an intrinsic order
- **Numeric variables** → validated against feature-specific min/max ranges; out-of-range values set to NaN.
- **Time-derived features** → extracted from `meta_incision` (month, weekday, daytime window, hour).
- **Hematocrit** → values > 1 rescaled by ÷100 before numeric validation [0, 1].

## Subsets

### Emergency Data

The structure of the data itself stays the same. However, this subset uses additional inclusion and exclusion criteria:

* **Inclusion Criteria:**
  * Urgency ≤ 3 → N4 and N5 were defined as elective and were consequently excluded
* **Exclusion Criteria:**
  * Column-Completeness < 50 % → columns with completeness below 50 % were excluded.
  * Case-Completeness < 50 % → cases with completeness below 50 % were excluded.

Note: Some meta or target columns may be excluded in subsets.
